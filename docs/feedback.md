# Feedback de usuarios - DondeEntreno

Este documento registra comentarios y observaciones obtenidas a partir de pruebas manuales realizadas por personas cercanas al proyecto.

El objetivo de esta etapa es validar si la plataforma se entiende, si es fácil de usar y si la experiencia inicial resulta clara para usuarios reales.

## Link probado

```text
https://donde-entreno-web.vercel.app/
```

## Personas que probaron el MVP

```text
- Pareja del desarrollador.
- Amigos cercanos.
- Usuarios no técnicos.
```

## Preguntas usadas para pedir feedback

```text
1. ¿Se entiende de qué trata la plataforma?
2. ¿Te parece fácil de usar?
3. ¿Pudiste encontrar la sección Explorar?
4. ¿Pudiste usar los filtros?
5. ¿Pudiste abrir el detalle de una actividad?
6. ¿Probaste la web desde el celular?
7. ¿Algo se vio roto, raro o confuso?
8. ¿Qué le agregarías?
```

## Observaciones generales

```text
- La plataforma fue compartida con usuarios reales para una primera prueba informal.
- La prueba busca validar claridad, navegación, diseño responsive y comprensión general de la idea.
- El feedback recibido servirá para definir próximas mejoras del MVP.
```

## Mejoras candidatas

* Mejorar textos de presentación si algún usuario no entiende rápido la idea.
* Revisar filtros si algún usuario se confunde al usarlos.
* Mejorar diseño mobile si se detecta algún problema visual.
* Agregar más actividades de prueba para que la plataforma se vea más completa.
* Mejorar la página Publicar para explicar mejor cómo funcionaría el alta de actividades.
* Agregar mensajes de carga o estados vacíos más claros.
* Agregar referencias, valoraciones o información de contacto más visible en el detalle.
